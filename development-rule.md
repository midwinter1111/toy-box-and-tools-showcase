# 🔨開発ルール🔨

- リポジトリは常に<https://github.com/midwinter1111/toy-box-and-tools-showcase>を参照するようにしてください。

- 開発作業のたびにリポジトリの最新実装状況を確認し、古いコードでの議論をしないようにしてください。

- 要件や特別な指示が無い限りは関係ない箇所のコードを修正したり、リファクタリングをしたりしないでください。修正したい場合はあくまで修正案として別コメントで提示してください。ただし、要件を満たすためにレイアウト等を修正することは可とします。

- 画面ファイルはviews配下に.vueファイルを作成し、templateタグおよびstyleタグを含んだファイルとしてください。

- 画面で用いられるロジックはcomposables配下の.composable.tsファイルを作成し、scriptタグで定義されるべきロジック処理を記述するようにしてください。つまり、.vueファイルに必要以上なロジックを持たせないようにしてください。

- 画面ファイル（.vue）とロジックファイル（.composable.ts）はセットで考えてください。例えば、SamplePageという画面を作成する場合、画面ファイルはSamplePage.vue, ロジックファイルはSamplePage.composable.tsとしてください。また、.composable.ts内で定義するconst値はuseSamplePageLogicのような命名とし一貫してどの画面で利用されるものかを明確にしてください。

- 本開発においてはSPAを前提とし、画面遷移はrouter配下のindex.tsにてAPP_PATHおよびroutesを定義することで、各画面からrouter.push()の引数に指定する形で画面遷移するようにしてください。つまり、各画面のロジックにむやみやたらとrouteの定義を記述しないようにしてください。

- 画面は特別な指示がない限り、PCとスマホ両方でデザインが崩れないよう、レスポンシブデザインの思想で設計をしてください。

- package.jsonに記述されていないパッケージを導入するのが効果的な場合、いきなり導入するのではなくメリット/デメリットや対応工数を踏まえて導入すべきかをレビュアーに相談し、レビュアーの判断のもと導入をしてください。ただし、導入する際はpnpm add pinia@4.1.2のようにバージョン固定でのインストールを前提とし、pnpm installコマンドのような常に最新バージョンを参照するようなインストールは脆弱性の観点で禁止とします。
